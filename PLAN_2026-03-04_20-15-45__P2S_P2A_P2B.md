# 详细规划：P2-S / P2-A / P2-B

时间戳：2026-03-04_20-15-45
对应路线：Phase 2 下一阶段

## 1. P2-S 升级不中断底座

## 1.1 目标

- 升级过程中不中断用户关键任务。
- 新版本异常时自动回滚。

## 1.2 设计

- 双槽位：`active` 与 `candidate`
- 升级流程：
  1. 部署 candidate
  2. 健康检查（API/DB/memory/plugin）
  3. 灰度流量切换
  4. 观测窗口（错误率/延迟）
  5. 全量切换或自动回滚
- 任务保障：
  - `running` 任务优先在 active 完成
  - 必要时恢复到 checkpoint 在 active 重跑

## 1.3 关键接口

- `POST /upgrade/prepare`
- `POST /upgrade/healthcheck`
- `POST /upgrade/promote`
- `POST /upgrade/rollback`
- `GET /upgrade/status`

## 1.4 验收

- 升级失败 5 分钟内自动回退。
- 升级期间任务完成率不显著下降。

## 2. P2-A 多 Agent 并发可视化管理

## 2.1 目标

- 用户可同时看到多个 agent 的并发任务进度与阻塞状态。

## 2.2 设计

- 多 agent 模型：
  - `agent_id`, `role`, `status`, `capacity`
- 调度映射：
  - `task -> subtask -> agent`
- 可视化：
  - 泳道视图（按 agent）
  - DAG 视图（按 task）
  - 关键路径高亮
  - 队列/并发配额监控

## 2.3 关键接口

- `GET /agents`
- `POST /agents/register`
- `POST /tasks/{task_id}/assign`
- `GET /dashboard/agents`
- `GET /dashboard/tasks/{task_id}/dag`

## 2.4 验收

- 支持同时显示 >= 20 个活跃任务的状态。
- 支持按 agent/status/user 过滤。

## 3. P2-B AIRI 风格皮套人交互层

## 3.1 目标

- 用户通过角色化界面完成任务交互与下发。

## 3.2 设计

- Persona 层：
  - 角色设定、语气、行为偏好
- Avatar 层：
  - 渲染适配（Live2D/VRM）
  - 状态驱动表情/动作
- 语音层：
  - ASR（输入）
  - TTS（输出）
- 联动：
  - 任务状态变化推送角色状态机

## 3.3 关键接口

- `GET /persona/profile`
- `POST /persona/profile`
- `POST /voice/transcribe`
- `POST /voice/speak`
- `GET /avatar/state`

## 3.4 验收

- 角色态与任务态一致（事件驱动同步）。
- 角色模块异常不影响任务执行主链路。

## 4. 风险与缓解

- 风险：可视化复杂度导致前端性能下降。
  - 缓解：分页 + 虚拟列表 + 采样渲染。
- 风险：升级切换导致状态不一致。
  - 缓解：升级锁 + 事务化状态转移 + 回放校验。
- 风险：皮套人集成拖慢主路径。
  - 缓解：严格分层，角色层只消费事件流。

## 5. Sprint 建议

- Sprint 1（1 周）：P2-S 最小可用（手动 promote/rollback）
- Sprint 2（1-2 周）：P2-A 基础看板（agents + task dag）
- Sprint 3（1-2 周）：P2-B 最小角色层（persona + basic avatar + tts）
